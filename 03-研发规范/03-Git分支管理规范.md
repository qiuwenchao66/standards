# Git分支管理规范

## 1. 目的
统一分支模型和协作方式，降低集成冲突和发布风险。

## 2. 分支类型
- `main`：生产可发布分支
- `develop`：日常集成分支
- `feature/*`：功能开发分支
- `release/*`：发布准备分支
- `hotfix/*`：线上紧急修复分支

## 3. 使用要求
- 功能开发从 `develop` 拉取 `feature` 分支
- 发布前从 `develop` 创建 `release` 分支
- 紧急修复从 `main` 创建 `hotfix` 分支
- 合并完成后按流程回合到必要分支

## 4. 合并要求
- 禁止直接向 `main` 提交未经评审代码
- 合并前应完成自测、冲突处理和必要同步
- 合并应保留清晰变更记录

## 5. 分支命名建议
- `feature/user-login`
- `release/1.2.0`
- `hotfix/fix-payment-timeout`

## 6. 清理要求
- 已合并分支应定期删除
- 长期分支应定期同步主干，避免偏离过大
