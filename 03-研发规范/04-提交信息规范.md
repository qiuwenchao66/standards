# 提交信息规范

## 1. 目的
统一提交记录格式，便于审计、回溯和自动化生成变更说明。

## 2. 建议格式
`type(scope): summary`

示例：
- `feat(auth): add sms login support`
- `fix(order): handle duplicate callback`
- `docs(api): update error code section`

## 3. type 建议
- `feat`：新功能
- `fix`：缺陷修复
- `refactor`：重构
- `docs`：文档修改
- `test`：测试相关
- `chore`：构建、脚本、依赖等杂项

## 4. 编写要求
- 首行简洁明确，说明本次变更核心内容
- 一个提交尽量只表达一类变更
- 禁止使用无意义说明，如 `update`、`modify`

## 5. 关联要求
- 重要提交建议关联需求编号、缺陷编号或任务编号
- 发布修复类提交建议在描述中补充影响范围
