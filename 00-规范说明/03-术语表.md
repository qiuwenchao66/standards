# 术语表

## 1. 目的
统一规范库中的常用术语，减少理解偏差。

## 2. 常用术语
| 术语 | 说明 |
|---|---|
| 规范库 | 统一维护的通用规则集合 |
| 项目模板 | 新项目启动时可复制使用的标准文档模板 |
| 项目文档 | 项目实施过程中实际填写和维护的文档 |
| 代码评审 | 对代码变更进行质量、风险和规范检查的活动 |
| 质量门禁 | 在提测、发布或交付前必须满足的质量条件 |
| 配置项 | 系统运行依赖的参数、密钥、地址等可配置内容 |
| 灰度发布 | 分批次、分范围逐步放量的发布方式 |
| 回滚 | 发布异常时恢复到上一可用版本的操作 |
| 敏感数据 | 涉及身份、账户、隐私、业务机密的数据 |

## 3. 使用要求
- 项目如使用本表未覆盖的新术语，应在项目文档中补充定义
- 关键缩写应在首次出现处给出全称
