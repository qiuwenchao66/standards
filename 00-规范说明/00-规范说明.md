# 规范说明

## 1. 目的
建立统一的软件开发规范体系，明确项目执行标准、协作边界和交付要求，保障项目过程可控、结果可复用。

## 2. 适用范围
- 所有新建软件项目
- 所有进入维护期的持续迭代项目
- 涉及需求、设计、开发、测试、发布、运维的相关成员

## 3. 使用原则
- 通用事项遵循规范库
- 项目特例单独说明并保留审批记录
- 同一事项如存在多份规范，以最新发布版本为准
- 规范不替代项目设计文档，项目文档负责落地本规范

## 4. 文档层级
1. 规范库：统一规则和最低要求
2. 项目模板：项目启动时复制使用的文档模板
3. 项目文档：实际项目中的需求、设计、测试、上线材料

## 5. 管理要求
- 每份规范应具备编号、版本、作者、审批人、发布日期
- 规范更新需附带变更记录
- 规范执行情况可纳入评审、审计或项目复盘

## 6. 建议元数据
- 文档编号
- 文档名称
- 当前版本
- 状态（草稿/生效/废止）
- 作者
- 审核人
- 生效日期

## 7. 附录
可在项目启动时引用本规范说明作为总入口，并在项目 README 或项目章程中明确引用版本。
