# 软件开发规范库

## 1. 目标
本规范库用于统一软件项目在研发、数据、测试、质量、安全、发布和运维过程中的执行标准，降低协作成本，提高交付质量。

## 2. 适用范围
- 适用于公司内部新建或迭代的软件项目
- 适用于产品、研发、测试、运维、项目管理等角色
- 适用于 Web、移动端、后台服务、数据服务等常见项目类型

## 3. 使用方式
- 新项目启动时默认引用本规范库
- 项目可在此基础上补充项目级细则，但不得与通用规范冲突
- 如存在例外情况，应通过评审并形成书面记录

## 4. 维护原则
- 规范库统一维护，按版本迭代
- 规范变更应记录变更原因、影响范围和生效时间
- 已发布项目如需升级规范，应评估兼容性和实施成本

## 5. 目录说明
- `00-规范说明`：规范总则、编号和版本规则、术语
- `03-研发规范`：编码、命名、Git、提交、评审、日志、异常、配置、接口
- `04-数据规范`：数据库、SQL、数据字典、数据安全
- `05-测试与质量规范`：测试、缺陷、测试用例、质量门禁
- `06-安全与运维规范`：安全开发、发布、部署、监控、故障处理
