# SQL编写规范

## 1. 目的
统一 SQL 书写和执行要求，降低性能问题和数据风险。

## 2. 基本要求
- SQL 应清晰可读，关键字建议统一大小写风格
- 查询字段应按需选择，避免无必要的 `select *`
- 更新和删除语句必须带明确条件

## 3. 性能要求
- 高风险查询应先评估执行计划
- 避免在大表上无索引条件扫描
- 批量操作应控制单次影响范围

## 4. 安全要求
- 动态参数必须参数化，避免 SQL 注入
- 生产环境禁止直接执行未经评审的大批量变更

## 5. 审核要求
- 上线 SQL 应经过测试验证
- 重要 SQL 变更应保留回滚脚本或回退方案
